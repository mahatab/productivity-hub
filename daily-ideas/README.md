# Daily Ideas - Built Apps & Prototypes

This folder contains daily app ideas that have been **actually built** as prototypes.

## Purpose

Instead of just suggesting ideas in the morning briefing, Rudro:
1. Generates an idea based on news, trends, or m2labs focus
2. **Builds a working prototype** (HTML/CSS/JS app or site)
3. Takes a screenshot
4. Includes both in the daily email

## Structure

Each idea gets its own folder:
```
YYYY-MM-DD-idea-name/
├── index.html          # The app/site
├── screenshot.png      # Visual proof
└── README.md          # Idea description & notes
```

## Philosophy (m2labs)

Every idea must be:
- ✅ **Focused** - Solves one specific problem
- ✅ **Simple** - Clean, minimal UI
- ✅ **Actionable** - Could become a real product
- ✅ **Built** - Not just a concept, but a working demo

## Examples of Good Ideas

- **News focus timer** - Distraction-free reading with countdown
- **Audio story queue** - Simple playlist for Bengali stories
- **Task priority sorter** - Drag-drop urgent vs important
- **Meeting cost calculator** - Real-time $ cost of meetings
- **One-tap reminders** - Single button for common reminders

## Examples of Bad Ideas

- ❌ "Social media platform" (too broad)
- ❌ "AI that does everything" (not focused)
- ❌ "Complex dashboard" (against m2labs philosophy)

## Workflow

1. **Generate idea** - Based on news/trends/pain points
2. **Build prototype** - Quick HTML/CSS/JS implementation
3. **Screenshot** - Visual proof of concept
4. **Share in email** - Morning briefing includes screenshot
5. **Evaluate** - Could this become a real m2labs product?

*Built with focus. One idea at a time.*
